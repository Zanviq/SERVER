# TwoEMS v2 — 멀티유저 개인 홈서버 워크스페이스 구현 계획

> **For agentic workers:** 본 계획은 단계(Phase)별로 구현하고 **기능 하나당 커밋**한다. 각 단계 끝에 독립적으로 테스트 가능한 산출물이 나와야 한다. 체크박스(`- [ ]`)로 추적.

**Goal:** 비밀번호 세션 로그인 + 멀티유저 개인/공통 파일공간 위에, 파일관리·노트(마크다운/위키링크/그래프)·캘린더·ReAct AI 비서를 통합한 자작 홈서버 워크스페이스를 만든다.

**Architecture:** FastAPI 단일 게이트웨이(세션 인증 미들웨어로 전 API 보호) + React/Tailwind SPA(react-router, 좌측 사이드바, 다크/라이트). 저장소는 `STORAGE_ROOT/common`(공통)과 `STORAGE_ROOT/users/<uid>`(개인)로 스코프 분리. AI는 Plant-Counselor식 ReAct(스킬 레지스트리+오케스트레이터)로 사용자 스코프 내에서만 동작.

**Tech Stack:** Backend: FastAPI, itsdangerous(세션 서명), psutil, google-generativeai, google-api-python-client(선택). Frontend: React 18 + TS + Vite + TailwindCSS v3 + react-router-dom + FullCalendar + react-force-graph-2d + react-markdown + Zustand.

## Global Constraints

- **인증 필수**: `/api/auth/login` 과 정적 자산을 제외한 **모든** API는 유효 세션 필요. 미인증 → 401.
- **세션 TTL = `SESSION_TTL_SECONDS`(기본 3600초)**. 만료 시 자동 로그아웃. 남은 시간 노출.
- **계정은 `.env`의 `AUTH_USERS`(JSON)에서만** 정의. 코드/DB에 사용자 생성 기능 없음.
- **저장소 스코프 격리**: 한 사용자는 다른 사용자의 개인폴더에 (UI로도 AI로도) 접근 불가. 모든 경로 해석은 `(common | 본인 personal)` 루트 기준 + `..` 차단.
- **보안 기본값**: CORS는 명시적 오리진만(자격증명 사용). 에러 응답은 `DEBUG=false`면 일반 메시지(서버 경로/예외 비노출), 상세는 서버 로그에만.
- **민감정보 필터**: AI로 전송되는 파일명/내용은 민감 키워드 사전 필터.
- **ARM64 호환**(라즈베리파이5). Python 3.11 컨테이너, Node 20.
- **디자인**: Plant-Counselor 팔레트/토큰(올리브그린, 따뜻한 크림/다크그린), CSS 변수 + `data-theme`. 폰트 Pretendard + JetBrains Mono.

## 저장소/디렉토리 구조

```
STORAGE_ROOT/
├── common/                      # 공통 파일 공간 (모든 사용자 공유)
└── users/
    └── <username>/
        ├── files/               # 개인 파일
        ├── notes/               # 마크다운 노트 (.md, 위키링크)
        ├── calendar/events.json # 내부 캘린더 이벤트
        ├── settings.json        # 개인 설정
        └── ai/logs/             # AI 대화 턴 로그
```

가상 경로 표기: `common/...` 또는 `me/...`(=본인 personal). API는 `scope`(`common`|`me`) + `path`를 받아 실제 디스크 경로로 해석.

## Phase 1 — 인증(세션 로그인) + 보안 하드닝

**목표:** `.env` 계정으로 로그인, 1시간 세션, 전 API 보호, 남은시간 표시, 보안리뷰 5건 해결.

**Files:**
- Create: `backend/auth.py`, `backend/routers/auth.py`
- Modify: `backend/config.py`(AUTH_USERS/SESSION_*/DEBUG/CORS), `backend/main.py`(미들웨어/예외/CORS), `backend/routers/files.py`·`ai.py`(에러 일반화, 민감필터)
- Test: `backend/test_smoke.py`(로그인/보호/만료)

**Interfaces (Produces):**
- `auth.issue_token(username) -> str`, `auth.verify_token(token) -> dict|None`(`{username, exp}`)
- 의존성 `require_session(request) -> SessionUser(username, display_name, expires_at)`; 미인증 401.
- `POST /api/auth/login {username,password} -> {ok, expires_at}` (+ HttpOnly 쿠키 `tw_session`)
- `POST /api/auth/logout -> {ok}`; `GET /api/auth/session -> {username, display_name, expires_at, remaining}`

**핵심 설계:**
- 토큰: `itsdangerous.TimestampSigner` 또는 서명된 JSON(`{u, exp}`) + `SESSION_SECRET`. 만료는 exp로 강제.
- 보호: FastAPI `app` 전역 `dependencies=[Depends(require_session)]` 또는 라우터별. 로그인/health만 예외.
- 쿠키 HttpOnly+SameSite=Lax(+Secure when https). 프론트 fetch `credentials:"include"`.
- 보안리뷰 반영: CORS 명시 오리진+`allow_credentials=True`(와일드카드 금지, 시작 시 검증); 예외/업로드 에러는 DEBUG=false면 일반 메시지.

**Steps:**
- [ ] config에 AUTH_USERS(JSON 파싱), SESSION_SECRET/TTL, DEBUG 추가; 시작 시 SESSION_SECRET 없으면 경고/거부.
- [ ] auth.py: 서명 토큰 발급/검증 + require_session 의존성.
- [ ] routers/auth.py: login(비번 상수시간 비교)/logout/session.
- [ ] main.py: 전역 인증 의존성, CORS 수정, 예외 핸들러 DEBUG 게이트.
- [ ] files/ai 에러 메시지 일반화 + AI 검색 민감필터.
- [ ] 테스트: 미인증 401 / 로그인 성공 / 잘못된 비번 401 / 만료 토큰 401.
- [ ] 커밋: `feat(auth): .env 기반 세션 로그인(1h) + 전역 보호 + 보안 하드닝`

## Phase 8 — AI: ReAct 오케스트레이터 + 스킬 + 스케줄링

**목표:** Plant-Counselor식 ReAct. 스킬을 연속 실행하며 계획(think)·스케줄(캘린더/마감) 수립. 사용자 스코프 내에서만 파일/노트/캘린더 접근(보안).

**Files:** Create `backend/ai/{skill_base,skill_registry,orchestrator,prompt_builder}.py`, `backend/ai/skills/*`; Modify `backend/routers/ai.py`(SSE chat). Frontend `components/ai/AssistantPanel.tsx`(스텝 표시).

**스킬(최소):** `think`, `list_files`, `read_file`, `search_files`, `write_note`, `read_note`, `list_notes`, `create_calendar_event`, `list_calendar_events`, `summarize_file`. 각 스킬은 `SkillContext`(user, scope roots)만으로 동작 → **다른 사용자 접근 불가**.

**Interfaces (Produces):**
- `SkillBase{name,description,parameters,run(args,ctx)->SkillResult}`; `SkillResult{ok,message,data}`; `SkillContext{user, common_root, personal_root, services}`.
- `SkillRegistry.build_catalog()/dispatch(name,args,ctx)`.
- `orchestrator.run(user, message, history) -> SSE`(event: tool_call/tool_result/token/done). max_steps=`AI_MAX_STEPS`.
- `POST /api/ai/chat`(SSE) — 스코프는 항상 요청 사용자.

**보안:** SkillContext의 루트는 require_session의 user로만 구성. 스킬은 그 외 경로 접근 불가. 민감 키워드 필터 유지.

**Steps:**
- [ ] skill_base/registry + think + read-only 스킬(list/read/search) + 테스트.
- [ ] mutating 스킬(write_note, create_calendar_event) + 확인 게이트(옵션).
